## 1. 产品愿景

RepoPilot 是一个 **AI 驱动的开源项目学习平台**。它帮助用户管理 GitHub Star 项目，通过内置的多 Agent 协作系统深度解读开源项目，借助知识图谱可视化项目间的关联，让用户在多项目融合中高效学习。

**核心差异化**: RepoPilot 不是一个简单的 GitHub Star 管理器。它的 AI Agent 系统由六个专业 Agent 组成——快速分析师、深度导师、学习规划师、分类管家、笔记助手、对话管家——各司其职，能够跨项目交叉分析、拆解架构、生成学习路径，帮助用户从「收藏了但没学」变成「系统性掌握」。

**一句话定位**: 你的 AI 开源项目学习领航员。

## 2. 目标用户

| 用户画像  | 特征                | 痛点              | 需求               |
| ----- | ----------------- | --------------- | ---------------- |
| 技术学习者 | 大量 Star 项目但缺乏系统学习 | 收藏容易学习难，项目间缺乏联系 | 交叉学习、进度追踪、知识关联   |
| 全栈开发者 | 关注前后端多个领域的项目      | 项目太多记不住，技术选型困难  | 快速对比、架构拆解、适用场景分析 |
| 开源爱好者 | 深度参与开源社区          | 需要了解同类项目差异      | 横向对比、技术演进、生态分析   |
| 技术管理者 | 团队技术栈评估           | 缺乏项目全景视图        | 图谱总览、分类统计、推荐导入   |

## 3. 功能模块

### 3.1 用户系统 (User System)

| 功能        | 优先级 | 描述                               |
| --------- | --- | -------------------------------- |
| 注册/登录     | P0  | 用户名+密码注册，JWT Token 认证            |
| 修改密码      | P0  | 旧密码验证 + 新密码设置                    |

| 用户设置      | P1  | 主题、字体、缩放等个性化配置                   |

### 3.2 项目管理 (Project Management)

| 功能               | 优先级 | 描述                       |
| ---------------- | --- | ------------------------ |
| GitHub Star 批量导入 | P0  | 拉取 GitHub Star 列表，一键批量导入 |
| 手动添加项目           | P0  | 填写名称、URL、分类、标签、描述等       |

| 列表/卡片双视图         | P1  | 切换项目展示模式                 |

### 3.3 AI Agent 系统 (核心差异化)

#### 3.3.1 系统定位

RepoPilot 的 Agent 系统是由多个专业 Agent 协同工作的开源项目学习专家网络（Scout / Mentor / Navigator / Curator / Scribe / Hub），遵循 BYOK、深度优先、主动反问、有记忆、可配置五大核心原则。

> Agent 系统的完整设计（核心原则详述、角色行为、记忆系统、工具集、交互协议、降级策略、个性定义）请参阅 **[AGENT_PRD.md](./AGENT_PRD.md)**（F5-06 修正：同目录相对路径）。

#### 3.3.2 Agent 角色总览

| Agent      | 代号            | 职责                      | 触发场景                      |
| ---------- | ------------- | ----------------------- | ------------------------- |
| 快速分析 Agent | **Scout**     | 快速概览项目的核心信息（30s 出结果）    | 导入项目时自动触发、项目卡片上的”快速分析”按钮  |
| 深度讲解 Agent | **Mentor**    | 逐层拆解项目，反问了解用户水平，定制化讲解   | 项目详情页、对话中提问“讲讲这个项目”       |
| 学习规划 Agent | **Navigator** | 基于用户知识图谱规划学习路径          | 设置页“学习规划”、对话中提问“帮我规划学习路线” |
| 分类文档 Agent | **Curator**   | 智能分类、生成项目文档、标签建议        | 导入时自动建议分类、手动触发“重新分类”      |
| 笔记助手 Agent | **Scribe**    | 辅助生成笔记大纲、总结、代码注释        | 笔记编辑器内的 AI 按钮             |
| 对话管家       | **Hub**       | 统一对话入口，路由用户问题到合适的 Agent | 全局对话窗口                    |

## 4. 非功能需求

### 4.1 性能

- 页面首屏加载 < 2s
- API 响应 < 500ms（不含 Agent 调用）
- Agent 流式输出首 token < 3s（Scout 快速分析 < 5s）
- 图谱渲染 500 节点 < 2s（目标）；最低验收 100 节点 < 2s（F5-04 分层定义）
- Agent 反问面板渲染 < 500ms
- 上下文组装 < 200ms

### 4.2 安全

- JWT Token 认证，Token 过期机制
- 密码 bcrypt 哈希存储
- HTTPS 传输加密（部署时）
- CSRF/XSS 防护
- API 速率限制（Agent 端点额外限流）
- 输入校验和 SQL 注入防护
- Agent 权限隔离（工具权限分级）
- API Key 加密存储（不明文、不暴露给前端）
- Prompt 注入防护

### 4.3 兼容性

- 桌面端: Windows 10+, macOS 12+（pywebview/Electron）
- Web 端: Chrome 90+, Firefox 90+, Safari 15+, Edge 90+
- 响应式布局: 最小宽度 900px

### 4.4 可维护性

- 前后端分离，模块边界清晰
- **代码覆盖率：** Service 层 ≥ 70%（核心安全模块 100%，业务模块 70%）
- 完善的 API 文档
- 统一的代码规范（ESLint/Ruff/Prettier）
- Agent 配置标准化（AGENT.md / SOUL.md / config.yaml），支持导入导出

## 5. 用户故事 (核心场景)

### US-01: 新用户入门

> 作为一名开发者，我注册 RepoPilot 后绑定 GitHub 账号，一键导入 200+ Star 项目。Curator Agent 自动为它们建议分类并弹出确认面板，我看到一个漂亮的知识图谱，发现我的 React 和 Vue 项目之间有很多相似的技术模式。

### US-02: 深度学习

> 我想学 Next.js，打开项目详情页，Mentor Agent 先弹出反问面板了解我的 React 和 Node.js 水平，然后为我定制讲解了 Next.js 的 App Router、Server Components、Data Fetching 三大核心模块。Mentor 告诉我它和我已有的 React、Express 项目有很多共通之处，并给出了对比分析。

### US-03: 交叉学习

> 我问 Agent "对比我库里的 Flask 和 FastAPI"，Hub 把问题路由给 Mentor。Mentor 从路由设计、中间件机制、异步支持、性能基准四个维度对比，并建议我先学 Flask 打基础再学 FastAPI。

### US-04: 学习规划

> 我告诉 Navigator "我想在 3 个月内成为全栈开发者"，Navigator 基于我的项目库和技术画像（JS 精通、Python 入门），规划了一条从 React 深入 → FastAPI 入门 → 全栈融合的 10 周学习路径，每个阶段都有明确的里程碑。

### US-05: 笔记与回顾

> 学习 React Hooks 时，我在项目笔记里记录自己的理解。Scribe Agent 帮我补充了几个容易踩坑的点和最佳实践。一个月后我回来看笔记，快速回忆起关键知识。

### US-06: 图谱探索

> 在图谱中我看到 D3.js 和 Three.js 之间有一条连线，点进去发现它们都基于 WebGL 渲染。Navigator Agent 解释了两者的不同定位，建议我根据可视化需求选择。

### US-07: 无 Key 降级使用

> 我没有配置 API Key，但仍然可以导入 GitHub Star、使用关键词规则分类、查看图谱和手动管理项目。Agent 相关功能显示友好的提示引导我去设置页配置 Key。

## 7. 发布计划 (Release Plan)

### 7.1 版本策略

RepoPilot 采用 **v1.0 单版本完整发布** 策略。不再拆分 v0.1 Foundation、v0.2 Insight、v0.3 Connect、v0.4 Scholar、v0.5 Memory、v0.6 Polish 等子版本。

**理由:**

- v1.0 是一次性完整交付的产品，包括用户系统、项目管理、分类系统、笔记系统、可视化图谱、Agent 系统（6 个 Agent）等所有 P0 功能和核心 P1 功能（裁剪清单详见 MVP_SCOPE.md §2.2）
- MVP_SCOPE.md 的功能范围即 v1.0 全部交付内容
- 子版本拆分会导致开发团队在路线图上反复犹豫、文档间互相矛盾

**后续版本路线（v1.0 之后）:**

| 版本  | 主题              | 范围                                                              |
| --- | --------------- | --------------------------------------------------------------- |
| v1.1 | 用户体验增强          | 文件上传头像、OAuth GitHub 绑定、JSON 导入导出、列表/卡片双视图、笔记搜索、笔记导出            |
| v1.2 | 数据可视化扩展        | 分类统计图、学习进度看板、时间线                                          |
| v1.3 | Agent 高级能力     | Agent 协作（Hub 编排多 Agent）、记忆系统完善、用户画像主动学习                         |
| v1.4 | 多端与生态          | Web 端独立部署、移动端适配、Skill/插件市场、MCP 集成                              |

### 7.3 质量门禁 (v1.0 发布前必须全部通过)

- 后端 Service 层单测覆盖率 ≥ 70%（核心模块 100%，业务模块 70%）
- 前后端 lint 0 errors、0 warnings
- TypeScript 编译通过，禁止使用 `any`
- 集成测试覆盖所有 MVP 端点
- E2E 测试覆盖 5 条 happy path（注册→导入→图谱→笔记→Agent 对话）
- 所有 🔴 严重审查问题修复
- 性能指标全部达标（首屏 < 2s、API P95 < 500ms、图谱 100 节点 < 2s、TF-IDF 200 项目 < 3s）
