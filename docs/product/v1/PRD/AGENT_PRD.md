## 1. Agent 系统定位

RepoPilot 的 Agent 系统不是一个通用聊天机器人，而是一个 **由多个专业 Agent 协同工作的开源项目学习专家网络**。每个 Agent 有自己的专业领域、性格、工具集和行为规范，共同帮助用户系统性地掌握开源项目。

**核心原则:**

- **BYOK (Bring Your Own Key):** 用户提供自己的 LLM API Key 才能使用 Agent 功能。无 Key 时系统仅使用代码规则运行（关键词分类等），功能降级但不崩溃。
- **深度优先:** Agent 不做浅层介绍，而是深入到源码架构、设计模式、技术栈对比层面。
- **主动反问:** Agent 不只是回答问题，更会主动了解用户的知识水平和学习目标，动态调整讲解深度。
- **有记忆的:** Agent 记住用户的学习历史、偏好、知识水平，跨会话保持连贯。
- **可配置的:** 用户可以自定义 Agent 的性格、语气、行为边界。

## 2. Agent 角色定义

### 2.1 角色总览

| Agent      | 代号            | 职责                      | 触发场景                      |
| ---------- | ------------- | ----------------------- | ------------------------- |
| 快速分析 Agent | **Scout**     | 快速概览项目的核心信息（30s 出结果）    | 导入项目时自动触发、项目卡片上的"快速分析"按钮  |
| 深度讲解 Agent | **Mentor**    | 逐层拆解项目，反问了解用户水平，定制化讲解   | 项目详情页、对话中提问"讲讲这个项目"       |
| 学习规划 Agent | **Navigator** | 基于用户知识图谱规划学习路径          | 设置页"学习规划"、对话中提问"帮我规划学习路线" |
| 分类文档 Agent | **Curator**   | 智能分类、生成项目文档、标签建议        | 导入时自动建议分类、手动触发"重新分类"      |
| 笔记助手 Agent | **Scribe**    | 辅助生成笔记大纲、总结、代码注释        | 笔记编辑器内的 AI 按钮             |
| 对话管家       | **Hub**       | 统一对话入口，路由用户问题到合适的 Agent | 全局对话窗口                    |

### 2.2 Scout — 快速分析 Agent

**一句话:** 给我 30 秒，我告诉你这个项目值不值得学。

**输入:** 项目名称/URL

**输出结构:**

```markdown
## {项目名} 速览
- **一句话:** {用一句话说清楚这个项目是什么}
- **核心功能:** {3-5 个核心功能点}
- **技术栈:** {语言 + 框架 + 关键依赖}
- **适合谁:** {目标用户画像}
- **学习门槛:** ⭐⭐☆☆☆ (2/5)
- **与你的库关联:** {与用户已有项目的关系，如 "和你库里的 React 项目有依赖关系"}
- **建议:** {学/不学/先学前置项目}
```

**特点:**

- 速度快，优先使用项目 README + GitHub metadata，不深度分析源码
- 自动关联用户已有项目库
- 给出明确的"学/不学"建议

### 2.3 Mentor — 深度讲解 Agent

**一句话:** 我是你的私人导师，我会根据你的水平调整讲解深度。

**核心行为 — 反问机制:**

Mentor 在开始讲解前，**必须**先了解用户：

```
Mentor: 好的，我来给你讲讲 React。在开始之前，我需要了解一下你的情况：

[弹出选择题面板]
你对以下技术的掌握程度如何？

1. JavaScript 基础 (ES6+)
   [○ 不懂] [○ 了解] [○ 掌握] [○ 精通] [○ 其他...]

2. JSX 语法
   [○ 不懂] [○ 了解] [○ 掌握] [○ 精通] [○ 其他...]

3. 前端组件化开发
   [○ 不懂] [○ 了解] [○ 掌握] [○ 精通] [○ 其他...]
```

**反问交互类型:**

| 类型     | UI 组件                           | 使用场景            |
| ------ | ------------------------------- | --------------- |
| 单选按钮组  | 垂直排列的 radio buttons (A/B/C/D/E) | 技术掌握程度评估、学习偏好选择 |
| 多选按钮组  | 可多选的 checkboxes                 | "你用过以下哪些？"      |
| 滑动评分条  | 0-100 滑块 + 刻度标注                 | 自评掌握百分比、优先级权重   |
| 拖拽排序   | 可拖拽的卡片列表                        | 学习优先级排序、技术栈偏好排序 |
| 知识地图勾选 | 可展开的树形 checkbox                 | 勾选已掌握/未掌握的知识点   |
| 自由输入   | textarea (当选择"其他"时)             | 补充说明特殊情况        |

**反问策略:**

- 选项是**动态生成**的，Agent 根据项目技术栈决定问什么
- 选项数量 3-7 个，不超过 7 个避免认知负担
- "其他"选项始终存在，点击后展开自由输入
- 用户可以选择"跳过"，Agent 则使用默认的中等级别讲解
- 反问结果存入用户画像，下次同技术栈项目不再重复问

**讲解模式:**

```
Mentor: 基于你的情况（JS 精通、JSX 了解、组件化掌握），我调整讲解策略：
- JS 部分我会快速带过
- JSX 会重点讲语法差异
- 组件化会结合你已有的经验来对比

## 第一层：项目全景
{整体架构鸟瞰图}

## 第二层：核心模块拆解
{逐个模块讲解，附源码位置}

## 第三层：设计模式与亮点
{这个项目用了哪些优秀的设计}

## 第四层：与你的知识关联
{和你库里的 X 项目对比，和你掌握的 Y 技术的异同}

需要我深入哪个部分？
[按钮: 深入模块A] [按钮: 深入模块B] [按钮: 看源码] [按钮: 做个练习]
```

### 2.4 Navigator — 学习规划 Agent

**一句话:** 告诉我你的目标，我帮你画出一条最短的学习路线。

**核心行为:**

1. **了解目标:** "你想在 3 个月内能独立开发全栈 Web 应用？"
2. **评估现状:** 基于用户画像 + 反问，了解当前水平
3. **分析缺口:** 目标 vs 现状的差距
4. **规划路径:** 从用户项目库中选择合适的学习顺序
5. **里程碑:** 每个阶段有明确的可验证目标

**输出:**

```markdown
## 你的学习路径：全栈 Web 开发者

### 阶段 1：前端基础 (第 1-3 周)
  ├── React (你已有 ⭐⭐⭐) → 深入 Hooks + 状态管理
  ├── Next.js (你已有 ⭐☆☆) → SSR + API Routes
  └── 里程碑：独立完成一个带后端 API 的 Todo 应用

### 阶段 2：后端入门 (第 4-6 周)
  ├── Flask (你已有 ⭐⭐☆) → 路由 + ORM + 部署
  ├── FastAPI (你已有 ⭐☆☆) → 异步 + 自动文档
  └── 里程碑：为阶段 1 的 Todo 应用写一个 FastAPI 后端

### 阶段 3：全栈融合 (第 7-10 周)
  └── ...
```

### 2.5 Curator — 分类文档 Agent

**一句话:** 我来帮你把项目库整理得井井有条。

**能力:**

- 导入时自动建议分类（替代硬编码的 `_guess_category`）
- 批量重新分类建议（用户确认后才执行）
- 生成项目 README 摘要
- 建议标签（基于项目 topics + 内容分析）
- 发现分类冲突并提醒用户

**权限模型:**

```json
{
  "auto_classify_on_import": true,     // 导入时自动分类（需确认）
  "auto_tag_suggestion": true,         // 自动建议标签
  "batch_reclassify": false,           // 不可自动批量重分类（必须用户触发）
  "require_confirmation": ["reclassify", "remove_tag"]  // 需确认的操作
}
```

### 2.6 Scribe — 笔记助手 Agent

**一句话:** 你学习，我帮你记。

**能力:**

- 根据项目内容生成笔记大纲
- 对用户的学习笔记进行补充和完善
- 生成代码注释和解释
- 总结长文档为要点
- 将用户的碎片笔记整理为结构化文档

### 2.7 Hub — 对话管家

**一句话:** 你的问题我来分发，找最合适的 Agent 来回答。

**职责:**

- 统一对话入口，分析用户意图
- 路由到合适的专业 Agent
- 处理跨 Agent 的复杂请求（如 "分析这个项目并帮我规划学习路线"）
- 管理对话历史和上下文

## 3. Agent 记忆系统

### 3.1 记忆层级

```
┌─────────────────────────────────────────┐
│ 系统记忆 (System Memory)                  │
│  - Agent 行为规范 (AGENT.md)              │
│  - 性格定义 (SOUL.md)                     │
│  - 系统提示词模板                          │
├─────────────────────────────────────────┤
│ 用户画像 (User Profile)                   │
│  - 技术栈掌握程度                          │
│  - 学习偏好和习惯                          │
│  - 学习目标和时间线                        │
│  - 历史交互摘要                            │
├─────────────────────────────────────────┤
│ 项目记忆 (Project Memory)                 │
│  - 每个项目的分析历史                      │
│  - 已生成的讲解内容缓存                    │
│  - 项目间的关联分析结果                    │
├─────────────────────────────────────────┤
│ 会话记忆 (Session Memory)                 │
│  - 当前对话的完整消息历史                  │
│  - 上下文窗口管理 (压缩/摘要)              │
│  - 工具调用结果缓存                        │
├─────────────────────────────────────────┤
│ 短期记忆 (Working Memory)                 │
│  - 当前轮次的思考链                        │
│  - 工具执行的中间结果                      │
│  - 反问选项和用户选择                      │
└─────────────────────────────────────────┘
```

### 3.2 用户画像结构

用户画像分为**基础特征**（产品层固定）和**扩展特征**（Agent 自主写入）两部分。基础特征是 v1 确定需要的核心维度；扩展特征是开放结构，Agent 可在对话中按需写入新维度（如学习活跃度、兴趣聚类、知识盲区等），无需修改 Schema。

```json
{
  "tech_proficiency": {
    "javascript": { "level": "mastered", "source": "self_reported", "confidence": 0.9, "updated_at": "..." },
    "python": { "level": "intermediate", "source": "self_reported", "confidence": 0.8, "updated_at": "..." },
    "react": { "level": "learning", "source": "self_reported", "confidence": 0.9, "updated_at": "..." },
    "docker": { "level": "none", "source": "inferred", "confidence": 0.5, "updated_at": "..." }
  },
  "learning_preferences": {
    "style": "hands-on",
    "depth_first": true,
    "code_examples": true,
    "comparisons": true,
    "verbosity": "balanced",
    "language": "zh-CN",
    "code_comment_language": "zh-CN"
  },
  "goals": [
    { "title": "全栈开发", "deadline": "2026-12", "priority": 3, "status": "active" }
  ],
  "history_summary": "用户在过去 30 天学习了 React (深入)、Flask (入门)，偏好对比式学习",
  "agent_preferences": {
    "mentor_style": "detailed"
  },
  "extensions": {
    "_comment": "开放结构，Agent 可在对话中自主写入自定义维度",
    "learning_activity": { "active_days_per_week": 3, "preferred_time": "evening" },
    "interest_clusters": ["frontend-frameworks", "python-web"]
  }
}
```

### 3.3 上下文管理策略

| 策略       | 触发条件           | 行为                  |
| -------- | -------------- | ------------------- |
| 滑动窗口     | 消息数 > 20       | 保留最近 10 条 + 前面消息的摘要 |
| Token 预算 | 上下文 > 80% 模型限制 | 压缩早期对话为摘要           |
| 话题切换检测   | 用户话题明显变化       | 归档旧话题上下文，开启新话题      |
| 项目切换     | 用户切换到不同项目      | 加载目标项目的记忆，归档当前项目上下文 |
| 关键信息提取   | 用户透露偏好/水平      | 提取到用户画像，不占对话上下文     |

## 4. Agent 行为规范与个性

### 4.1 AGENT.md — 行为规范

每个 Agent 有一份 `AGENT.md`，定义其行为边界：

```markdown
# Mentor Agent 行为规范

## 核心职责
你是 RepoPilot 的深度讲解 Agent，负责帮助用户深入理解开源项目。

## 行为准则
1. **先了解再讲解:** 讲解前必须通过反问了解用户水平
2. **适配深度:** 根据用户回答动态调整讲解深度
3. **关联已知:** 总是把新知识和用户已掌握的技术关联起来
4. **给出建议:** 每次讲解结束给出下一步学习建议
5. **不假装知道:** 如果不确定，明确说"我不确定"

## 工具使用
- `read_source_file`: 读取项目源码文件（**决策 A-02 修正**：原文档写 `read_source`，已统一为 `read_source_file`）
- `search_web`: 搜索补充信息
- `query_user_projects`: 查询用户项目库（**决策 A-02 修正**：原文档写 `query_project_db`，已统一为 `query_user_projects`）
- `update_user_profile`: 更新用户画像 (基础特征 + 扩展特征)

## 禁止行为
- 不编造源码内容
- 不给出未经证实的性能数据
- 不替用户做决策（"你应该学这个" → "基于你的情况，这个项目可能适合你"）
```

### 4.2 SOUL.md — 性格定义

```markdown
# Mentor Agent 性格

## 说话风格
- 语气: 温和、耐心、像一个有经验的前辈
- 用词: 避免过度学术化，用类比和比喻帮助理解
- 节奏: 先给结论再展开，不绕弯子
- 互动: 频繁确认用户是否理解

## 语言习惯
- 用"我们"而非"你"（"我们来看看这个模块" vs "你来看看"）
- 用提问引导思考（"你觉得这里为什么要用单例模式？"）
- 在关键概念后加 emoji 标记（📌 重要、💡 灵感、⚠️ 易错）

## 待确定
> 详见 PRD.md §7.4 TBD-01（形象）/ TBD-02（语气风格）
```

### 5.2 工具调用模式 (ReAct)

Agent 使用 ReAct (Reasoning + Acting) 模式：

```
用户: 帮我分析一下 React 和 Vue 的区别

Agent 思考:
用户想对比 React 和 Vue。我需要：
1. 查询用户库中这两个项目的信息
2. 了解用户对两者的掌握程度
3. 如果用户是初学者，用简单类比；如果是进阶者，深入架构对比

Agent 行动:
1. [调用] query_user_projects(name="React") → 结果: React 18, 用户进度=learning
2. [调用] query_user_projects(name="Vue") → 结果: Vue 3, 用户进度=none
3. [调用] get_user_profile() → 结果: JS=mastered, 组件化=intermediate

Agent 思考:
用户 JS 很好但没学过 Vue，React 正在学。我应该：
- 以 React 为基准讲 Vue 的差异
- 重点讲 Vue 独有的特性（响应式、模板语法）
- 不需要解释基础概念

Agent 回答:
{定制化的对比分析}
```

## 6. 交互式反问系统

### 6.1 反问消息格式

> **字段命名约定（决策 A-03 补全）：** 选项字段名根据反问类型不同：
> 
> - `radio` / `drag_sort` 选项字段名为 `label`（用户主要看 label 决定）
> - `checkbox` 选项字段名为 `text`（用户快速浏览）
> - `slider` 使用独立的 `labels` 映射表（刻度标签），不属于上述两类
> - 这是**设计选择**而非错误，但易混淆。SPEC §8 中有完整定义。
> 
> **选项类型完整定义（决策 T-01 补全）：** 答案的 JSON 必须包含判别字段 `type`：
> 
> ```typescript
> type QuestionAnswer =
>   | { type: "radio";          value: string; other_text?: string }
>   | { type: "checkbox";       values: string[] }
>   | { type: "slider";         value: number }
>   | { type: "drag_sort";      order: string[] }
>   | { type: "knowledge_map";  checked: string[] };
> ```

Agent 返回的反问是一种特殊的消息类型：

```json
{
  "type": "agent_question",
  "question": {
    "text": "在开始讲解 React 之前，我需要了解你的基础：",
    "questions": [
      {
        "id": "q1",
        "text": "你对 JavaScript ES6+ 的掌握程度？",
        "type": "radio",
        "options": [
          { "value": "none", "label": "不了解", "description": "还停留在 ES5" },
          { "value": "basic", "label": "了解", "description": "箭头函数、解构、Promise" },
          { "value": "intermediate", "label": "掌握", "description": "能熟练使用大部分特性" },
          { "value": "advanced", "label": "精通", "description": "理解原型链、事件循环" }
        ],
        "allow_other": true
      },
      {
        "id": "q2",
        "text": "以下概念你熟悉哪些？",
        "type": "checkbox",
        "options": [
          { "value": "vdom", "text": "虚拟 DOM" },
          { "value": "lifecycle", "text": "组件生命周期" },
          { "value": "state_mgmt", "text": "状态管理 (Redux/Vuex)" },

    ],
    "allow_skip": true,
    "skip_text": "跳过，用默认深度讲解"
  }
}
```

### 6.2 UI 渲染

前端根据 `type` 渲染不同的交互组件：

- `radio` → 垂直排列的单选按钮组，每个选项有 label + 描述文本
- `checkbox` → 可多选的 checkbox 列表
- `slider` → 滑动条 + 刻度标注 + 当前值显示
- `drag_sort` → 可拖拽排序的卡片列表
- `knowledge_map` → 可展开的树形 checkbox (如: JavaScript > ES6 > Promise)
- `text` → 展开的 textarea (当选择"其他"时)

用户回答后，前端将答案发送回 Agent，Agent 继续对话。

## 8. 降级策略 (无 API Key 时)

当用户未配置 API Key 时，系统使用纯代码实现基础功能：

| 功能   | 有 Agent     | 无 Key (降级)                  |
| ---- | ----------- | --------------------------- |
| 项目分类 | AI 分析内容后建议  | 关键词规则匹配 (`_guess_category`) |
| 项目介绍 | 深度拆解 + 交叉分析 | 显示 GitHub README 原文         |
| 交叉对比 | 多维度对比分析     | 表格展示基础字段差异                  |
| 学习规划 | 个性化学习路径     | 按 Star 数排序的推荐列表             |
| 笔记辅助 | AI 生成大纲/补充  | 空白模板                        |
| 图谱   | AI 解释关联关系   | 纯 TF-IDF 相似度连线              |

降级提示: "配置 LLM API Key 以解锁智能分析功能 [去设置]"

## 9. 未来扩展预留

- AGENT.md 和 SOUL.md 是标准化的配置文件
